11/25/2019 Release Notes: The current release is version 1.3.0

Skill Flow Builder Editor download: Win or Mac

Release Changes

The following features have changed or improved.

  • Editor
    • We have improved the Editor's ability to handle large (500 scene) files. Tip: use multiple .abc files to structure a skill that is over 500 scenes. This can also make it easier to organize content.
    • You probably won't notice, but "Writer Mode", represented by the pencil icon, has been removed. Problems with this mode were slowing the Editor, so we removed it in favor of the Editor handling larger skills.
    • Added undo and redo commands to work with guided mode in the Editor. Window size and position are also remembered when you reopen the editor.
  • Core Modules
    • Coming soon: major updates to technical documentation, coverage includes SFB Build Life Cycle, use of custom extensions, and other advanced use cases.
    • Customer Request: You can now use multiple stages: if you want to say create a beta stage and a prod stage for your skill, this often involves creating two separate skills in the Developer Console. To make this easier, you can use the --stage argument with the deploy command when using the sfb-cli.
    • Customer Request: With very minimal effort, you can now get better insights into how your users are navigating your SFB skills. The tooling supports various levels of configuration; from plug and play all the way to full BI integration. Check out the SFB Metrics Extension
    • Added debug flag to command line simulator to enable quiet mode. Makes it easier to see your project specific logging.
    • Updated core extension classes to make them easier to subclass for your own extensions.
    • Snippets now work in localized content.
    • Fixed an issue where recap/reprompt responses were erroneously combining their output.
    • Updated handling of AMAZON.StopIntent to return "shouldEndSession" flag as true to conform to new Alexa policy.

Known Issues

Please be aware of the following known issues with this release, we are actively working to fix these in an upcoming release:

  • Alexa soundbank URLs of the format 'soundbank://soundlibrary/...' will not play in simulator mode.
